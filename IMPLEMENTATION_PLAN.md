# Магазин горнолыжного и сноубордического снаряжения — план реализации

## Цели и база
- Витрина без онлайн-оплаты на старте; сбор заказов и контактов.
- RU/KG интерфейс и контент; фоллбек: если нет перевода, показываем другой язык.
- Валюта: KGS. Доставка: СДЭК + самовывоз (1 точка), запрос стоимости доставки через e-mail/Telegram.

## Технологический подход
- **Стек:** Next.js (App Router) + Headless CMS (Strapi) в Docker Compose; Postgres для CMS, S3-совместимое хранилище (minio/локально) для медиа.
- **Локализация:** i18n в Next.js, коллекции в CMS с полями для RU/KG; ручной переключатель языка.
- **Развёртывание:** docker-compose.yml с сервисами `frontend`, `cms`, `db`, `storage`; инструкции запуска на Ugreen NAS.

## Каталог и модель данных
- Категории и подкатегории согласно `categories.txt`; отдельные разделы для alpine/split/snowboard bindings; Climbing/Treking без подразделов.
- Товар: базовые поля (название, описание, бренд, цена в KGS, изображения), теги.
- Вариации: размер/ростовка (при необходимости), цена общая для товара, без фото на вариацию и без контроля остатков (покупка не блокируется при 0).
- Фильтры: цена, размер/ростовка, бренд. Таблицы соответствий размеров (EU/US/UK/см) как статические страницы RU/KG.

## Покупательский путь
1) Каталог с фильтрами и языковым переключателем.
2) Карточка товара: общие фото, выбор вариации размера/ростовки, цена в KGS, краткие характеристики.
3) Корзина (сохранение для гостя в localStorage) + оформление без онлайн-оплаты: контакты, адрес/комментарий.
4) Информация о доставке: ссылка/кнопка «узнать стоимость» с e-mail `djdrewsha@gmail.com` и Telegram `Andrei_outdoor_stuff`.
5) Личный кабинет: регистрация/логин, история заказов с деталями позиций (название + размер, количество, цена), повтор оплаты не требуется.

## Админка и роли (CMS)
- Роли: `admin` (полный доступ), `manager` (товары, цены, остатки, промокоды, заказы без настройки системы).
- CRUD для категорий, товаров, вариаций, промокодов.
- Промокоды: процентная скидка, даты действия, опциональный лимит активаций.
- Импорт ассортимента: шаблон Google Sheets (CSV-экспорт) с колонками категории, название RU/KG, бренд, цена, вариации; загрузка CSV через CMS с базовой валидацией формата.
- Мультиязычные статические страницы: возврат, контакты, условия доставки.

## Статические страницы
- Политика возврата RU/KG (подготовить черновик текста).
- Контакты: адрес самовывоза (12 Microdistrict, 3/2, Chuy, Bishkek, KYRGYZSTAN), часы 9–18 ежедневно, карта (встраиваемая), e-mail и Telegram.

## Хостинг и эксплуатация
- Контейнеризация под малую нагрузку NAS; документация по запуску/обновлению.
- Резервные копии БД/медиа (скрипт/cron dump + sync в файловое хранилище NAS).

## Этапы
1) Настройка репо, Docker Compose, базовый фронт + Strapi с моделями и локализацией.
2) Каталог/товары/вариации, фильтры, мультиязычный UI, корзина и checkout без оплаты.
3) Личный кабинет и история заказов; промокоды (процент).
4) Импорт из CSV (шаблон Google Sheets) и админ-ролевая модель.
5) Статические страницы (контакты, возврат, доставка), таблицы размеров, финальные инструкции деплоя.
